Capitulo 1: Qué es CSS

Introducción a CSS

CSS (Cascading Style Sheets) es un lenguaje basado en reglas que definen grupos de estilos.

Elementos de CSS: selector (de etiquetas HTML), declaración contiene propiedad (colores, fuentes, etc) y valor (para los colores, fuentes, etc); el conjunto de todo esto se llama regla.

Las reglas de CSS se usan para dar una estética agradable a los sitios web.

Estructurar CSS

Hoja de estilo externa

El CSS estará escrito en un archivo externo al documento HTML. Solo debemos crear un archivo nuevo con la extensión .css y ponerlo en algun lugar de la carpeta.

En el HTML debemos usar el elemento link para insertar el archivo css. Tiene el atributo href en donde va la ruta del archivo css.

Con los selectores de CSS elegimos los elementos a los que le aplicaremos estilo.

Es el metodo más usado, pero existen otros:

  • Hoja de estilo interna: Funciona desde el propio documento html, dentro del head, usamos la etiqueta style y dentro de esta usamos selectores con sus reglas para aplicar css.
  • Estilo en linea: Usamos el atributo style y los valores de éste, serán las reglas (propiedad y valor) de CSS. Debemos separar las propiedades con punto y coma.

Selectores:

Hay diferentes formas de seleccionar elementos de HTML al cual le aplicamos estilos. Cada regla tiene un selector y estos tienen propiedad (color, párrafos, etc) y valor(numérico/nombre).

Especificidad:

Suponemos una situación donde dos selectores o más usan un mismo elemento para aplicarle estilo.

Cascada y Especificidad:

Entro dos elementos con la misma especificidad se aplicará el estilo del ultimo elemento escrito.

El siguente se sobreescribirá por la cascada, dado que está escrito ultimo.

Entre dos elementos en el cual hay uno que es mas especifico que otro, ganara este por mas que esté escrito primero.

Propiedad y Valor:

Propiedad: Son características de estilo (font-size, width, etc).

Valor: Indica como quieres que cambien las características de estilo.

Declaración y regla: La declaración es un conjunto de propiedades y valores que están dentro de las llaves del código de CSS. La regla son los selectores junto a sus propiedades y valores.

Ejemplos de Propiedades:

font-size

Es la dimension de la letra, es mejor usar px para darle valor.

width

Especifica la anchura del area de contenido de un elemento, está dentro del padding, borde y margen. Tiene varios tipos de valores.

background-color

Define el color de fondo de un elemento o puede ser trasparente (predeterminado).

color

Selecciona el valor de color para textos y decoraciones. El valor debe ser un color uniforme, no puede ser gradient (degradado/tipo de imagen).

border

Define todos los bordes (width, color, style) en una sola regla.

Funciones

Se pueden realizar distintas operaciones matemáticas para valores dados.

calc():

Se usa para determinar valores de propiedades mediante cálculos.

transform:

Te permite cambiar el espacio de coordenadas del modelo visual de css. Los elementos modificarse con funciones como trasladar, rotar, escalar o sesgar; de acuerdo a los valores introducidos en ellas.

@rules

Las reglas se pronuncian "at rules", algunas son simples otras cuando se cumple una condición podemos brindar instrucciones en CSS. Un ejemplo simple es:

@import "styles2.css";

Importará una hoja de estilo hacia otra hoja.

Otra regla muy frecuente es @media, está tiene tiene una condición que si se cumple ejecuta el css indicado.

Ejemplo, la regla definirá un fondo azul cuando se agrende la ventana en 30em en más.

Propiedades abreviadas (shorthands)

Pueden establecer varios valores en una sola línea, por ejemplo: font, background, padding, border, margin, etc.

En el ejemplo, aplicará los valores en el sentido de las agujas del reloj: top, right, bottom y left si ponemos los 4 valores; o si ponemos dos valores, el primero va para top/bottom y el segundo para left/right.

padding: 10px 15px 15px 5px;

Su equivalencia es:

padding-top: 15px; padding-right: 15px; padding-bottom: 15px; padding-left: 5px;

Los valores pueden afectar las cuatro direcciones: arriba, derecha, abajo, izquierda en este sentido (si ponemos 4 valores).

Como funciona CSS

El navegador combina el documento HTML que tiene todo el contenido de la web con el archivo CSS que determinará el aspecto del sitio.

Procesa el documento en una serie de etapas, pero antes se debe conocer ciertos conceptos tecnicos y no muy importantes cuando empezas:

El navegador y el DOM son los actores principales que interactuán con el HTML y el CSS.

El DOM (Modelo de Objetos del Documento) es una API (Interfaz de Programación de Aplicaciones) que es software que puede funcionar, comunicarse e interactuar con otro software. Permite ejecutar código en el navegador para acceder e interactuar con cualquier nodo del documento.

Se usa para interactuar y representar documentos HTML o XML. El DOM es un modelo de documento que se carga en el navegador y representa al documento como un árbol de nodos. Los nodos pueden crearse, modificarse o moverse.

Un nodo (en DOM) es un punto único en el árbol de nodos, puede ser un documento mismo, elementos de HTML, texto (contenido) o comentarios que representa una parte del documento.

El DOM se estructura como un árbol, sus ramificaciones van a ser como padres e hijos en donde esta el contenido que programamos.

Cuando en el HTML incluimos un CSS, el navegador lee el HTML y creará un DOM a partir de este así puede leer las reglas de CSS y aplicarlas.

Cuando el navegador lee un documento:

  1. El navagador carga el HTML
  2. Convierte el HTML en un DOM (el modelo de objetos del documento es un arbol que tiene nodos -elementos que programamos-), represetará el documento en la memoria del ordenador.
  3. El navegador va a buscar la mayor parte de los recursos vinculados al documento HTML, como las imagenes, los videos, el CSS vinculado al documento. JavaScript aparecerá más adelante.
  4. El navegador analiza el CSS y ordena las diferentes cajas/elementos (nodos) segun la cascada y espeficificad para aplicar los estilos (este proceso se llama arbol de renderización).
  5. El arbol de renderización presenta la estructura en que los nodos deben aparecer despues de aplicarle las reglas.
  6. En la pantalla se muestra el aspecto visual de la página (esta etapa se denomina painting).

En las herramientas para desarrolladores te mostrará todos los elementos que forman el DOM, te ayudarán a diseñar, depurar y mantener el CSS porque en el DOM se encuentra el HTML y el CSS.

En las herramientas se muestran se muestran tres paneles muestran (DOM, Propiedades y calculos de cajas) para navegar por el DOM, correguirlo y testear las funcionalidades del documento.

Después de que el navegador halla cargado y analizado el HTML, creaŕa un árbol (DOM), despues cargará el CSS y analizará los estilos (propiedades, valores) y los ordenará (herencia, cascada, prioridad, etc) para al final mostrarlos en pantalla.

Los elementos span se muestran de color lima.Si el navegador encuentra una regla mal escrita o propiedades que no tiene soportada o valores que no entiende, no hace nada/no se aplica y salta a otra propiedad. Analiza línea por línea.

Cuando inspeccionamos el DOM si podemos ver los errores.

Hay un concepto de usar propiedades solo soportadas por navegadores antiguos, antes de las propiedades nuevas para navegadores modernos dado que la cascada de CSS (si hay dos propiedades para un mismo elemento, aplica la ultima o la más especifica) y el navegador lo que no entiende lo saltea. Así es como se da soporte para la mayoría de navegadores.

Más Propiedades

font-family

define una lista de fuentes o familias de fuentes, con un orden de prioridad, para utilizar en un elemento seleccionado. los valores se separan con comas (",") para indicar que son valores alternativos.

Los valores puede ser: nombre de familia (Times, Helvetica, etc), las tengan espacios se escriben entre comillas; puede haber varias. nombre genérico (serif, sans-serif) no van entre comillas; se usa para cuando ninguna de las fuentes este disponible, puede haber una y se escribe al final.

Cuando alguna fuente no tiene alguna propiedad, pasa a la siguiente.

border-bottom

Permite de definir de una vez todas las propiedades individuales que describen el color, estilo y ancho: border-bottom-color, border-bottom-style, y border-bottom-width

Se puede especificar los tres valores en cualquier orden y omitir hasta 2

font-weight

Especifica el grueso de la letra, algunos tipos de letra solo están disponibles en normal o bold por ejemplo.

Valores: normal (es 400), bold (700), lighter (menos grueso que su elemento padre), bolder (mas grueso que su elemento padre) entre 100 y 900 (para los tipos de letra que nos permiten flexibilidad, si no existe algún valor, se presentara el mas aproximado entre la escala normal-bold)

font-size

Determina el tamaño de la letra, puede alterar el aspecto de otra cosa al usarse para calcular la longitud de las unidades em y ex, si hay que usar este tipo de valores mejor usar px (no varia en función de las caracteristicas del dispositivo)

text-decoration

Se usa para establecer el formato de texto a subrayado (underline) y suprarrayado (overline), tachado (line-through) o parpadeo (blink). El subrayado y el suprarrayado son posicionados bajo el texto, mientras la línea a través de éste se posiciona por encima.

Es una forma reducida, y puede usar los valores de cada una de las propiedades de la forma larga: text-decoration-line, text-decoration-color, y text-decoration-style.

Las decoraciones también se aplican a los elementos hijos/descendientes que esten en la misma linea, para solucionar esto al elemento hijo le agregamos una decoración vaciá o le aplicamos otro tipo de decoración en una nueva regla.
